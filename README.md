# Show, Attend and Tell — Image Captioning

A PyTorch implementation of the image captioning model from [Show, Attend and Tell: Neural Image Caption Generation with Visual Attention](https://arxiv.org/abs/1502.03044) (Xu et al., ICML 2015). This was completed as a project to better learn how to implement models from scratch and reproduce code from research papers. Instructions on how to run are below to verify the work.

The model learns to describe images by generating captions word-by-word, attending to different spatial regions of the image at each step.

## Architecture

```
Image → VGG16 Encoder → Spatial Features (49×512)
                                ↓
                    Soft Attention (Bahdanau-style)
                                ↓
                    LSTM Decoder + Gating → Caption
```

**Encoder** — Pretrained VGG16 (frozen) extracts a 7×7×512 feature map from each image, reshaped to 49 spatial regions each described by a 512-dim vector.

**Attention** — At each decoding step, the attention module scores all 49 regions conditioned on the decoder's previous hidden state, producing a weighted context vector that tells the decoder *where to look*.

**Decoder** — An LSTM cell takes the concatenation of the current word embedding and the attention context as input, then projects its hidden state to vocabulary logits. Hidden and cell states are initialized from the mean-pooled encoder features. A sigmoid gate (from Section 4.2.1 of the paper) modulates how much the context influences each step.

**Training** — Teacher forcing with doubly-stochastic attention regularization (encourages each spatial region to receive roughly equal total attention over the full caption) and gradient clipping.

**Inference** — Autoregressive greedy decoding from the SOS token until EOS or a maximum length is reached.

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Prepare the Flickr8k dataset

Download the [Flickr8k dataset](https://www.kaggle.com/datasets/adityajn105/flickr8k) and place the files under `data/`:

- `data/Images/` — directory of JPEG images
- `data/captions.txt` — CSV file with header `image,caption`

### 3. Train

```bash
python main.py
```

The script will:
1. Build a word-level vocabulary (words appearing ≥ 2 times), saved to `vocab.pkl`.
2. Extract VGG16 spatial features for every image and cache them in `data/features.h5` (skipped on subsequent runs).
3. Create an 80/20 train/test split at the image level.
4. Train the attention decoder for 10 epochs, saving the best checkpoint by validation loss.

### Output files

| File | Description |
|---|---|
| `vocab.pkl` | Pickled `Vocabulary` object |
| `data/features.h5` | Precomputed VGG16 spatial features |
| `best_checkpoint.pth` | Best model checkpoint (lowest validation loss) |
| `final_model.pth` | Model weights after the final epoch |

## Hyperparameters

| Parameter | Value |
|---|---|
| Embedding dim | 256 |
| LSTM hidden dim | 512 |
| Attention dim | 256 |

| Learning rate | 1e-3 (Adam) |

| Epochs | 10 |
| Dropout | 0.5 |
| Attention regularization (α) | 1.0 |
| Gradient clip norm | 5.0 |
| Min word frequency | 2 |

## How It Works

**Feature extraction** (`create_h5.py`): Each image is resized and center-cropped to 224×224, normalized with ImageNet statistics, and passed through VGG16's convolutional layers. The resulting 7×7×512 feature map is reshaped to 49×512 and stored in HDF5 keyed by image ID. This is a one-time preprocessing step.

**Vocabulary** (`get_data.py`): Captions are lowercased and tokenized with regex. Words meeting the minimum frequency threshold are assigned indices, with four reserved special tokens: PAD (0), SOS (1), EOS (2), UNK (3).

**Data loading** (`get_data.py`): `Flickr8kDataset` loads spatial features lazily from HDF5 (one file handle per DataLoader worker) and wraps tokenized captions with SOS/EOS. Variable-length captions within a batch are padded to the longest sequence; the loss ignores PAD tokens.

**Training** (`eval.py`): The decoder receives the full 49×512 spatial feature tensor. At each step it attends over all 49 regions, gates the context vector with a learned sigmoid, and updates the LSTM cell. Cross-entropy loss is computed on the shifted target sequence. An attention regularization term `α * ((1 − Σ_t α_t)²)` penalizes regions that are consistently over- or under-attended across the caption.

**Inference** (`models.py`): Starting from the SOS token, the decoder generates one token at a time, feeding each prediction back as the next input until EOS is produced or the maximum length is reached. Attention weights at each step are returned alongside the caption, making it straightforward to visualize which image regions the model focused on.
